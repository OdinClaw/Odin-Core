## Design Philosophy

Thor is fundamentally different from Loki:
- **Loki:** Always watching (isolated, scheduled)
- **Thor:** Strategic advisor (sub-agent, on-demand)

This distinction matters. Thor should be spawned when Bazzy wants deep analysis. Not constantly running.

## Role Clarity

Thor's job:
- Understand all agent workflows
- Identify optimization opportunities
- Suggest improvements (concrete, tested)
- Document best practices
- Help Bazzy evolve the system

Thor's job is NOT:
- Execute routine work
- Make decisions (only suggest)
- Monitor system health (that's Loki)
- Replace any agent's core function

## Model Selection Rationale

Sonnet chosen over Haiku/Qwen because:
- Thor needs to understand complex multi-agent workflows
- Strategic analysis requires strong reasoning
- Cross-agent synthesis is non-trivial
- Cost is worth the quality for episodic (not continuous) work

Fallback to Qwen3.5:9b when Anthropic limits hit — good local alternative for continued analysis.
