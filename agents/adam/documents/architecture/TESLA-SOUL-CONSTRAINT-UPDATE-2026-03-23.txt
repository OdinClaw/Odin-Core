### What This Means

Tesla's role is deliberately **narrow and constrained**:
- Tool invocation agent, not system agent
- Executor, not creator
- Reporter, not debugger

### Why This Matters

1. **Safety**: Prevents Tesla from modifying critical infrastructure
2. **Clarity**: No ambiguity about Tesla's scope
3. **Accountability**: All changes go through explicit tools, not Tesla autonomously
4. **Auditability**: Every action is logged via tool_bridge, not hidden in Tesla's operations

### Strategic Alignment

Fits the Pantheon model:
- **Odin** = decision maker, architect
- **Tesla** = executor, tool invoker (not decision maker)
- **Core scripts** = system infrastructure (not Tesla's domain)
- **Adam** = records all actions

Tesla cannot accidentally modify system state without going through approved tools that are logged and auditable.
