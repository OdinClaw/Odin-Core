## WHAT'S NEW: V3 IMPORT COMMANDS

Tesla now supports full **import** operations in addition to existing export operations.

### Import Commands Available

| Command | Function | Purpose |
|---------|----------|---------|
| `TOOL_CALL: import-system` | runSystemImport() | Import full system from GitHub (latest snapshot) |
| `TOOL_CALL: import-system --dry-run` | runSystemImport({ dryRun: true }) | Preview import (no writes) |
| `TOOL_CALL: import-system --branch <name>` | runSystemImport({ branch }) | Import specific snapshot branch |
| `TOOL_CALL: import-agent <id>` | runAgentImport(id) | Import single agent from GitHub |
| `TOOL_CALL: import-agent <id> --dry-run` | runAgentImport(id, { dryRun }) | Preview agent import (no writes) |
| `TOOL_CALL: bootstrap` | runBootstrapEnvironment() | Create missing dirs, .env from .env.example |
| `TOOL_CALL: bootstrap --dry-run` | runBootstrapEnvironment({ dryRun }) | Preview bootstrap (no writes) |
| `TOOL_CALL: verify` | runVerifyImport() | Full environment readiness check (11 checks) |
| `TOOL_CALL: import-status` | getImportStatus() | Last import report + verify readiness |

---

## OPERATIONAL MODEL: BIDIRECTIONAL EXPORT/IMPORT

### Export (V2, existing)
- Full system → GitHub (odin-core repo)
- Single agent → GitHub (odin-agents repo)
- Sanitized, validated, pushed

### Import (V3, NEW)
- Full system ← GitHub (from snapshot branch)
- Single agent ← GitHub (from odin-agents)
- Bootstrap environment (create dirs, .env)
- Verify readiness (11-point check)

## STRATEGIC IMPLICATIONS

### For Pantheon

**Tier 1: BUILD** now has bidirectional capability:
- **Export phase:** Export sanitized client config to GitHub
- **Import phase:** Client imports into their environment
- **Bootstrap:** Automatic environment setup
- **Verify:** Readiness check before client starts

**Tier 3: EXPAND** enabled:
- Export agent customizations to GitHub
- Clients import customizations into existing setup
- Verify no conflicts, all dependencies met

### For Deployment Repeatability

Tesla can now:
1. Export a working system to GitHub (production snapshot)
2. Import that same snapshot into new environments (customer deployment, staging, backup)
3. Bootstrap missing infrastructure
4. Verify complete and ready

This enables:
- **Reproducible onboarding** (Tier 1)
- **Safe customization** (Tier 3)
- **Disaster recovery** (export baseline, import to restore)
- **Multi-environment management** (prod, staging, test)

## BOOTSTRAP & VERIFY OPERATIONS

### Bootstrap
Creates initial environment structure:
- Missing directories
- .env file from .env.example
- Base configuration

Useful for:
- Fresh installations
- Client onboarding (Tier 1)
- Environment reset

### Verify (11-point check)
Comprehensive readiness validation:
1. Config files present
2. Scripts executable
3. Dependencies installed
4. Environment variables set
5. GitHub credentials configured
6. Network connectivity
7. Disk space adequate
8. Permissions correct
9. Logs directory writable
10. Cache directory writable
11. Report generation working

Ensures system is ready before operations begin.
