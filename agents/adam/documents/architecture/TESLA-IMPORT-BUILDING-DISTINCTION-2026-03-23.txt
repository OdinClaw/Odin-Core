KEY DISTINCTION: IMPORT IS TOOL EXECUTION, NOT BUILDING

### The Problem Being Solved

Tesla's import capability could be misunderstood as "building" — which is forbidden.

Solution: Explicit definition of the boundary.

---

## CRITICAL DISTINCTION

### BUILDING (FORBIDDEN — Tesla NEVER does this):
- Creating new scripts or files
- Rewriting or modifying existing scripts
- Scaffolding infrastructure or directories by hand
- Altering codebase logic or structure
- Iterating through fixes autonomously

**Why forbidden:** Prevents Tesla from modifying system infrastructure without approval.

### IMPORT (ALLOWED — Tesla executes these on demand):
- Calling `runSystemImport()` to copy validated export package
- Calling `runAgentImport(id)` to copy validated agent export
- Calling `runBootstrapEnvironment()` to create missing directories + stub `.env`
- Calling `runVerifyImport()` to check environment readiness

**Why allowed:** These are pre-built functions; Tesla only invokes them via tool_bridge.

**CRITICAL STATEMENT:** "These scripts ALREADY EXIST and are DEPLOYED. Tesla calls them via the tool bridge. Tesla does NOT build them, scaffold them, or modify them. Calling a pre-built function is NOT building."
